# IP Hygiene Copyright Checks

Load this when triage Q2 or Q3 hits.

## Check

- Do not reproduce third-party prose, code, examples, fixtures, figures, diagrams, tables, schemas, XML/JSON/YAML samples, audio, or video verbatim or near-verbatim unless quotation context and licence allow it.
- Prefer original paraphrase with an in-prose source citation when a specific source influenced the wording.
- Treat structured spec tables and enumerations as protected structure in EU contexts; do not copy row order, grouping, or column design as a dataset.
- Code samples carry their source licence. Short canonical API-call snippets may be unprotectable ideas, but longer creative examples need rewrite, citation, and licence review.
- Describe APIs; do not transcribe substantial SDK source.
- Redrawn diagrams and figures can still be derivative works.
- Build sample files from scratch; do not copy spec-distributed examples.

## Remedies

- Near-verbatim match: paraphrase into original expression and cite the identifiable source.
- Permitted quotation: quote only the amount needed, name source and author when possible, and avoid substituting for the source.
- Unclear source, licence, or quotation basis: stop and ask.
